# Report


## Botanix CI/CD system

**Contributors:** [@RaitoBezarius](https://github.com/RaitoBezarius), [@nikstur](https://github.com/nikstur), [@WilliButz](https://github.com/WilliButz), [@Ma27](https://github.com/Ma27)

A new scalable build and evaluation system, taking inspiration from the Bazel ecosystem, so you can build large things without exhausting the RAM of your system or being tied to a single machine.

- Built a Remote Build Execution protobuf interface and CLI client (`tokybuild`).
- Server integration: source tree creation, S3 presigned URLs, evaluation ([PR #45](https://git.afnix.fr/botanix/botanix/pulls/45)).
- Connected Botanix to [evolive](https://git.lix.systems/the-distro/evolive) for remote evaluations, storing eval results into S3.
- Redesigned the scheduler to support remote eval/build use cases.
- Added canonical tarball uploads and an eventbus system streaming worker events back to clients.
- End-to-end VM testing of Botanix.
- `rustfmt` + `clippy` cleanups across the codebase; easier local-dev instructions.
- A provisioning systemd service for the local Garage S3 development setup.

## laut — build signature verification

**Contributors:** [@mschwaig](https://github.com/mschwaig), [@lbeierlieb](https://github.com/lbeierlieb), [@SomeoneSerge](https://someonex.net), [@Noi0103](https://github.com/Noi0103)

- Working on [signature verification](https://github.com/mschwaig/laut/blob/a6bdcccfcce6e267d49a6e3355ded4e580d61274/rust/src/trust_model_reasoner.rst) where multiple signers need to agree about an output.
- Core hard problem: aggregating information across dependency signatures so that a signer can only contribute to a derivation's trustworthiness once, even when the same derivation appears in multiple resolutions of the dependency graph.
- Drew and analyzed dependency trees to think through possible verification models; considered probability-based approaches.
- Implemented and [tested a verification algorithm](https://github.com/mschwaig/laut/compare/fix_verification).
- Made VM test setup work without network access (avoiding `__impure`).
